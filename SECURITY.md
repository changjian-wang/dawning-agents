# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please use [GitHub Security Advisories](https://github.com/changjian-wang/dawning-agents/security/advisories/new) to report vulnerabilities privately.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

| Severity | Acknowledgment | Fix Target |
|----------|---------------|------------|
| Critical | 24 hours      | 7 days     |
| High     | 48 hours      | 14 days    |
| Medium   | 48 hours      | 30 days    |
| Low      | 72 hours      | Next release |

### What We Will Do

1. Confirm receipt within the timeline above
2. Investigate and validate the report
3. Develop and test a fix
4. Release a patched version
5. Credit the reporter (unless anonymity is requested)

## Out of Scope

- Vulnerabilities in dependencies with existing public CVEs (please report to the upstream project)
- Non-security-related bugs (please use [GitHub Issues](https://github.com/changjian-wang/dawning-agents/issues))
- Denial of service attacks against demo/sample applications
- Social engineering attacks

## Disclosure Policy

We follow [coordinated disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure). Please allow us reasonable time to address the issue before public disclosure.
